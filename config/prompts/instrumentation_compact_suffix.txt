Compact mode (to avoid truncation):
- Generate AT MOST 40 events total.
- Prioritize: page_load, primary CTAs, navigation, critical widgets (forms, carousels/lists, banners).
- **Grouping:** Collapse duplicate granularity only when it is **one widget** in the PRD (one `element_unique_name`, combined payload). **Do not** collapse **per-card** or **per-row** requirements—use one event **type** with **item_name** / **item_id** / **item_rank** (0-indexed: first = 0) and repeated triggers in **trigger** and **notes**, not a new `element_unique_name` per card. **`item_type`** = vertical only (`diagnostics` | `pharma` | `consult` | `others`); include **`patient_id`** on diagnostics patient-scoped rows.
- **Clicks:** Every `element_clicked` row must include `element_cta` using a simple action word (`view`, `ask`, `share`, `back`, `search`, `expand`, `select`, `show_all`, etc.). Buttons still get **their own** `element_unique_name` (≠ section). Card/row taps keep the section name plus `item_*` / `destination`.
- Keep the same column meanings: **story** = snake_case key; **metrics** = analysis outcomes; **notes** = implementation context. For lists, **trigger** must reflect a deliberate per-row vs section choice (tools like CleverTap: entity-level breakdowns vs module-only reach); keep rationale ultra-short in **notes**.
- Never use `event_props_json`. Prefer `custom_column1_name` / `custom_column1_value`, `item_id` / `item_name`, `property_section_name`, `item_sub_type`, and paired `metric_str_N` / `val_str_N`.
- Escape hatch for ambiguous fields: if unsure which first-class field to use, add `additional_cols: <json_object>` as a single line inside `event_specific_payload`.
- Skip noisy passive views unless clearly required: static strips/messages, tab-strip impressions, button impressions, follow-up strip views, section impressions for shortcut/navigation modules, promo-widget impressions when click-only is enough, and per-row impressions for non-clickable tables.
- Keep `trigger`, `event_specific_payload`, and `notes` as concise as possible while remaining clear.
